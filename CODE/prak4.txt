Praktikum 4 – Membuat Data Warehouse dan Melakukan 
ETL (Extract, Transform, Load) 
Matakuliah: Teknologi Perekayasaan Data 
 
Kita akan membuat dua database transaksional sebagai dua data source. Source pertama 
(A) menggunakan PostgreSQL dan Source kedua menggunakan MySQL Workbench. Setiap 
Source menyimpan data operasional mentah (OLTP) dengan struktur: 
1. Tabel fact_sales menyimpan transaksi penjualan 
2. Tabel dim_product menyimpan data produk 
3. Tabel dim_store menyimpan data toko/cabang 
4. Tabel dim_date menyimpan referensi waktu.

Download dan install terlebih dahulu MySQL Community Server di

Membuat Database Data Warehouse dengan Surrogate Keys 
Prinsip: 
1. Semua dimensi menggunakan surrogate keys 
2. ID dari source menjadi business key 
3. Tambahkan source_system untuk tracebility 
 
CREATE DATABASE dwh; 
USE dwh; 
 
CREATE TABLE dim_product ( 
    product_key INT AUTO_INCREMENT PRIMARY KEY, 
    product_id INT, 
    product_name VARCHAR(255), 
    category VARCHAR(100), 
    source_system VARCHAR(10) 
); 
 
CREATE TABLE dim_store ( 
    store_key INT AUTO_INCREMENT PRIMARY KEY, 
    store_id INT, 
    store_name VARCHAR(255), 
    city VARCHAR(100), 
    source_system VARCHAR(10) 
); 
 
CREATE TABLE dim_date ( 
    date_key INT PRIMARY KEY, 
    date DATE, 
    month INT, 
    year INT 
); 
 
CREATE TABLE fact_sales ( 
    sales_key INT AUTO_INCREMENT PRIMARY KEY, 
    product_key INT, 
    store_key INT, 
    date_key INT, 
    quantity INT, 
    revenue FLOAT, 
    source_system VARCHAR(10) 
); 
 
Proses ETL (Extract, Transform, Load) dengan PySpark 
1. Install Library di Python 
pip install pyspark pandas sqlalchemy pymysql 
pip install mysql-connector-python psycopg2 pandas 
sqlalchemy 
 
2. Import dan Setup 
from pyspark.sql import SparkSession 
from pyspark.sql.functions import lit, col, when 
from pyspark.sql.window import Window 
from pyspark.sql.functions import row_number 
 
spark = SparkSession.builder \ 
    .appName("ETL Data Warehouse") \ 
    .getOrCreate() 
 
3. Extract dari Source A dan Source B 
# Source A 
url_a = "jdbc:mysql://localhost:3306/source_a" 
# Source B 
url_b = "jdbc:mysql://localhost:3306/source_b" 
 
properties = { 
    "user": "root", 
    "password": "password", 
    "driver": "com.mysql.cj.jdbc.Driver" 
} 
 
# Load tables 
tables = ["dim_product", "dim_store", "dim_date", 
"fact_sales"] 
 
dfs_a = {t: spark.read.jdbc(url=url_a, table=t, 
properties=properties) for t in tables} 
dfs_b = {t: spark.read.jdbc(url=url_b, table=t, 
properties=properties) for t in tables} 
 
4. TRANSFORM 
a. Tambahkan Source ID 
for t in tables: 
    dfs_a[t] = dfs_a[t].withColumn("source_system", lit("A")) 
    dfs_b[t] = dfs_b[t].withColumn("source_system", lit("B")) 
 
b. Union Data 
df_product = 
dfs_a["dim_product"].unionByName(dfs_b["dim_product"]) 
df_store   = dfs_a["dim_store"].unionByName(dfs_b["dim_store"]) 
df_date    = dfs_a["dim_date"].unionByName(dfs_b["dim_date"]) 
df_sales   = dfs_a["fact_sales"].unionByName(dfs_b["fact_sales"]) 
 
c. Cleaning dan Standarisasi 
df_product = df_product.withColumn( 
    "category", 
    when(col("category").isin("Makanan", "Food"), "Food") 
    .when(col("category").isin("Minuman", "Drink"), "Beverage") 
    .otherwise(col("category")) 
) 
 
d. Generate Surrogate Keys pada Dimensions 
# Product Key 
window_spec = Window.orderBy("product_id", "source_system") 
 
dim_product_final = df_product.dropDuplicates(["product_id", 
"source_system"]) \ 
withColumn("product_key", row_number().over(window_spec)) 
# Store Key 
window_spec = Window.orderBy("store_id", "source_system") 
dim_store_final = df_store.dropDuplicates(["store_id", 
"source_system"]) \ 
    .withColumn("store_key", row_number().over(window_spec)) 
 
# Date Key 
dim_date_final = df_date.dropDuplicates(["date_id"]) 
 
e. Mapping Fact → Dimension Lookup 
Join untuk ambil surrogate key 
fact_enriched = df_sales \ 
    .join(dim_product_final.select("product_id", 
"source_system", "product_key"), 
          ["product_id", "source_system"], "left") \ 
    .join(dim_store_final.select("store_id", "source_system", 
"store_key"), 
          ["store_id", "source_system"], "left") 
 
f. Final Fact Table 
fact_final = fact_enriched.select( 
    col("product_key"), 
    col("store_key"), 
    col("date_id").alias("date_key"), 
    col("quantity"), 
    col("revenue"), 
    col("source_system") 
) 
 
5. LOAD ke Data Warehouse 
url_dwh = "jdbc:mysql://localhost:3306/dwh" 
 
dim_product_final.write.jdbc(url=url_dwh, 
table="dim_product", mode="overwrite", 
properties=properties) 
dim_store_final.write.jdbc(url=url_dwh, table="dim_store", 
mode="overwrite", properties=properties) 
dim_date_final.write.jdbc(url=url_dwh, table="dim_date", 
mode="overwrite", properties=properties) 
fact_final.write.jdbc(url=url_dwh, table="fact_sales", 
mode="overwrite", properties=properties) 
 
6. Validasi 
fact_final.show() 
dim_product_final.show() 
 
 
7. Improvement 
a. Menggunakan Hash Key 
from pyspark.sql.functions import sha2, concat_ws 
 
df_product = df_product.withColumn( 
    "product_key", 
    sha2(concat_ws("_", "product_id", "source_system"), 256) 
) 
 
b. Incremental Load 
df_sales = df_sales.filter(col("date_id") > 20240101)
