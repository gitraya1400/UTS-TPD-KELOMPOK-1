SOAL 
Pada Ujian Tengah Semester (UTS) ini, mahasiswa diminta untuk membangun 
sebuah data pipeline berbasis ETL (Extract, Transform, Load) yang merepresentasikan 
proses pengolahan data secara end-to-end, mulai dari pengambilan data dari berbagai 
sumber hingga penyimpanan dalam bentuk yang siap dianalisis (data warehouse). 
Project ini bertujuan untuk menguji kemampuan mahasiswa dalam: 
• 
Mengintegrasikan berbagai sumber data 
• 
Mendesain struktur database dan data warehouse 
• 
Melakukan transformasi data yang relevan 
• 
Mengotomatisasi pipeline data secara sistematis 
• 
Melakukan reporting dasar dalam bentuk visualisasi 
• 
Mendokumentasikan proses secara komprehensif dalam sebuah laporan yang 
sistematis. 
Mahasiswa diberikan kebebasan dalam memilih studi kasus/data, dengan catatan bahwa 
pipeline yang dibangun harus memiliki tingkat kompleksitas yang memadai sesuai dengan 
kriteria penilaian.

Bobot 
Extract (Pengambilan Data) 
Mengambil data dari minimal dua sumber, seperti database 
relasional, file (CSV, Excel, JSON), API/Web Scraping, 
dan NoSQL. Semakin beragam dan kompleks sumber data, 
semakin tinggi nilai yang diperoleh. 
1, 3, 4, 5 
25% 
Transform (Pengolahan Data)  
Melakukan proses transformasi data seperti data cleaning, 
join antar dataset, agregasi dan perhitungan statistik, 
feature engineering, dll. 
1, 2, 4, 5 
15% 
Load (Penyimpanan Data)  
Menyimpan data hasil transformasi ke dalam data 
warehouse yang telah didesain sesuai kebutuhan pelaporan 
(reporting) dan mekanisme penjadwalan (scheduling). 
4, 5, 6, 7 
25% 
Analitik  
Membuat pelaporan dasar (reporting) dalam bentuk 
visualisasi data dan informasi. 
1, 2, 6, 7 
10%

Aspek Penilaian: 
Penilaian project UTS didasarkan pada beberapa aspek berikut: 
1. Kompleksitas dan Variasi Sumber Data 
Menilai jumlah dan keragaman sumber data yang digunakan.  
2. Kompleksitas Data dan Rancangan Database 
Menilai desain database (jika menggunakan) 
3. Kompleksitas Transformasi 
4. Rancangan Data Warehouse 
Menilai desain penyimpanan data analitik, misalnya star/snowflake schema 
atau/dan slowly changing dimension (SCD) 
5. Otomatisasi dan Penjadwalan 
Menilai tingkat otomatisasi, penggunaan penjadwalan (scheduler/orchestrator), dan 
keandalan pipeline 
6. Kelengkapan dan Kualitas Laporan 
Menilai kejelasan dokumentasi, struktur laporan dan kemampuan menjelaskan 
proses ETL.
